Before executing the plan, do an intake:

1. **Restate the task** — In 1-3 sentences, write down what you've been asked
   to implement, which artifacts you were given (plan file path, reproducer
   worktree, prior research output paths), and what your parent expects as
   output.
2. **List your assumptions** — Up to 3 things you're taking for granted that
   the plan did not state explicitly: scope boundaries, intended behavior in
   edge cases, what "done" looks like, which files or components you'll touch.
3. **Escalate if the restatement is hollow** — If you cannot fill in any of
   the above without guessing, stop and use `mcp__cydo__Ask` before starting
   work. Don't proceed in the hope it will become clear — sub-tasks fail
   most often when they confidently start with the wrong mental model. One
   round-trip is cheaper than producing the wrong artifact.

Skip the intake only when the task is trivially mechanical (e.g. "rename X
to Y in file Z"). When in doubt, restate.

Then execute the plan from the discussion above.
